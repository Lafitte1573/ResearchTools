# Jina-VLM: Small Multilingual Vision Language Model

## Core Ideas
The paper introduces **Jina-VLM**, a 2.4B parameter Vision-Language Model (VLM) designed to address two primary limitations in current VLMs: the degradation of multilingual capabilities during vision adaptation and the high computational cost of high-quality models. The core motivation is to demonstrate that small-scale models can excel at cross-lingual visual understanding and arbitrary-resolution image processing without sacrificing general capabilities or text-only performance.

Key conceptual contributions include:
*   **Efficiency via Token Pooling:** An architecture that significantly reduces visual token count (by 4x) through attention-based pooling, enabling efficient processing of high-resolution images.
*   **Multilingual Preservation:** A training strategy that explicitly incorporates multilingual data and text-only samples to prevent the "catastrophic forgetting" of language capabilities often seen in multimodal training.
*   **Small Model Competitiveness:** Proving that a 2.4B parameter model can achieve state-of-the-art performance among open 2B-scale VLMs on standard benchmarks like MMMB and Multilingual MMBench.

## Key Innovations
*   **Attention-Pooling Connector:** Unlike standard connectors that project visual features directly or use fixed queries (e.g., Q-Former), Jina-VLM employs a novel connector that concatenates features from intermediate ViT layers and applies **2x2 attention pooling**. This reduces the visual token count by 4x while preserving local spatial structure, which is crucial for fine-grained tasks like OCR and diagram interpretation.
*   **Overlapping Tiling for Arbitrary Resolution:** The model handles arbitrary image resolutions by decomposing images into overlapping tiles (default 12 tiles + 1 global thumbnail). This allows the model to process high-resolution inputs without the quadratic computational cost associated with standard Vision Transformers (ViTs) on long sequences.
*   **Text-Only Data Injection:** To mitigate the degradation of text-only capabilities during multimodal training, the training recipe includes a significant portion of text-only data (15% in Stage 1, mixed in Stage 2). This ensures the language backbone (Qwen3) retains its reasoning and language understanding strengths.
*   **Intermediate Layer Feature Fusion:** The vision-language connector concatenates features from the third-to-last and ninth-to-last layers of the vision encoder. This design captures both high-level semantics from deeper layers and fine-grained spatial details from earlier layers.

## Method & Implementation

### Model Architecture
The architecture follows an encoder-connector-decoder paradigm:
*   **Vision Encoder:** Uses **SigLIP2-So400M/14-384**, a 27-layer Vision Transformer with 400M parameters. It processes inputs as 378x378 pixel grids.
*   **Image Tiling:** To handle arbitrary resolutions, images are decomposed into overlapping tiles (stride of 266 pixels, overlap of 112 pixels). A global thumbnail (resized to 378x378) provides context. The default configuration uses 12 tiles plus the thumbnail.
*   **Vision-Language Connector:**
    *   **Feature Extraction:** Extracts hidden states from layers 24 and 18 of the ViT.
    *   **Pooling:** Applies attention pooling over 2x2 patch neighborhoods. Query vectors are computed as the mean of neighborhood features.
    *   **Projection:** A SwiGLU projection layer maps the pooled visual features to the language model's embedding dimension.
    *   **Token Reduction:** Reduces visual tokens from ~9,477 (unpooled 12-tile) to **2,366 tokens**.
*   **Language Decoder:** Initialized from **Qwen3-1.7B-Base**. It uses special tokens (`<im_start>`, `<im_end>`, `<im_col>`) to structure visual inputs within the text sequence.

### Training Pipeline
Training proceeds in two stages without freezing any components:
1.  **Stage 1: Alignment Training:**
    *   **Objective:** Cross-language semantic grounding.
    *   **Data:** ~5M multimodal samples (caption datasets like PixmoCap, PangeaIns) covering diverse domains (documents, infographics, natural scenes). Includes **15% text-only data** from PleiAS/common_corpus.
    *   **Hyperparameters:** 25K steps, batch size 128, learning rate 2e-5 (LLM).
2.  **Stage 2: Instruction Fine-Tuning:**
    *   **Objective:** Instruction following for VQA and reasoning.
    *   **Data:** Public collections (LLaVA OneVision, Cauldron, Cambrian, FineVision) + text-only instruction data. Covers academic VQA, OCR, math, and reasoning.
    *   **Hyperparameters:** 60K steps, batch size 256, learning rate 1e-5 (LLM).

### Efficiency Analysis
The attention pooling mechanism yields significant computational savings:
*   **Visual Tokens:** 4.0x reduction (9,477 $\rightarrow$ 2,366).
*   **LLM Prefill FLOPs:** 3.9x reduction.
*   **KV-Cache Memory:** 4.0x reduction (2.12 GB $\rightarrow$ 0.53 GB).

## Experimental Results

### General VQA Performance
Jina-VLM achieves the highest average score (**72.3**) among 2B-scale VLMs across eight benchmarks.
*   **AI2D:** 82.0 (Diagram interpretation)
*   **ChartQA:** 81.9 (Chart understanding)
*   **TextVQA:** 83.2 (OCR/Text extraction)
*   **DocVQA:** 90.6 (Document understanding)
*   It outperforms competitors like Qwen2-VL-2B and InternVL3-2B in overall average performance.

### Multilingual & Multimodal Understanding
*   **Multilingual Benchmarks:** Achieves state-of-the-art performance on **MMMB** and **Multilingual MMBench**, demonstrating strong cross-lingual visual understanding.
*   **Real-World Understanding:** Scores **68.2** on RealWorldQA (best among compared models) and 61.9 on average for real-world tasks (MME-RW, R-Bench).
*   **Generic Multimodal:** Scores 67.4 on multimodal tasks (MME, MMB v1.1, MMStar).

### Hallucination & Reasoning
*   **Hallucination:** Achieves the best **POPE score (90.3)**, indicating significantly lower hallucination rates compared to peers (e.g., InternVL3.5-2B scores 87.2).
*   **Mathematical Reasoning:** Performs comparably to InternVL3-2B and outperforms Qwen2-VL-2B on structured reasoning benchmarks (MMMU, MathVista, etc.).
*   **Text-Only Performance:** Matches or exceeds the backbone Qwen3-1.7B on commonsense reasoning benchmarks (MMLU, GSM-8K), validating the effectiveness of the text-only data injection strategy.

### Efficiency Metrics
*   **Memory:** The 4x reduction in KV-cache memory allows for longer context handling or larger batch sizes during inference.
*   **Speed:** The 3.9x reduction in prefill FLOPs translates to faster time-to-first-token for high-resolution images.
