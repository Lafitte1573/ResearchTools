# TinyChart: Efficient Chart Understanding with Visual Token Merging and Program-of-Thoughts Learning

## Core Ideas
The paper addresses the critical challenge of deploying high-performance chart understanding models in resource-constrained environments. While multimodal large language models (MLLMs) have shown remarkable capabilities, their large size (e.g., 13B parameters for ChartLlama) makes them impractical for many real-world applications, especially on consumer hardware with limited VRAM. The core idea is to build an *efficient* MLLM—TinyChart—that achieves state-of-the-art performance with only 3 billion parameters by tackling two key bottlenecks: (1) reducing the computational burden of numerical calculations through a novel Program-of-Thoughts (PoT) learning strategy, and (2) reducing the length of vision feature sequences for high-resolution charts via a parameter-free Visual Token Merging module.

The paper's central thesis is that these two innovations are synergistic: the PoT strategy provides the model with a powerful reasoning mechanism for numerical tasks, while the token merging enables the efficient encoding of high-resolution visual input required for accurate chart analysis. This combination allows TinyChart to outperform several 13B-parameter models on multiple benchmarks while achieving significantly higher inference throughput.

## Key Innovations
1.  **Visual Token Merging (VTM)**: A novel, parameter-free module inserted into each vision transformer layer. It operates by identifying and merging the most similar visual tokens across the entire feature map, thereby drastically reducing the sequence length without losing critical information. Unlike standard patch-based methods, VTM can merge non-adjacent tokens if they are sufficiently similar, making it highly effective for charts which often contain large, uniform areas of color and white space.
2.  **Program-of-Thoughts (PoT) Learning**: A training paradigm designed specifically for numerical reasoning in charts. Instead of directly predicting a final answer, the model is trained to generate executable Python code that performs the necessary calculations. This code is then executed by a Python interpreter to produce the final answer. The paper proposes two ways to construct the PoT dataset: Template-based PoT (using human-written templates) and GPT-based PoT (using a smaller LLM to generate diverse, complex programs).
3.  **Integrated Multitask Learning Framework**: The model is trained on a unified dataset that combines various chart understanding tasks (QA, text generation, table generation, instruction following), allowing it to learn a shared representation and adapt its behavior based on the specific task at hand.

## Method & Implementation
### Model Architecture
The architecture, as shown in Figure 2, follows the standard MLLM pipeline but with two key modifications:
1.  **Vision Transformer Encoder with VTM**: The encoder processes the high-resolution chart image, which is first divided into patches. The VTM module is integrated into each transformer layer, as illustrated in Figure 3. The process involves dividing the tokens into two sets, finding the most similar pairs across the sets, keeping the top-*r* most similar edges, and finally merging the endpoints via average pooling.
2.  **Vision-Language Connector**: This component projects the reduced vision features into the embedding space of the large language model (LLM). The paper implements this as a multi-layer perceptron with one hidden layer and GeLU activation.

### Visual Token Merging (VTM)
The VTM algorithm is detailed in Section 3.1.1 and Figure 3. Given an input image of resolution N×N and a patch size of P×P, the standard transformer would produce a feature sequence of length (N/P)². VTM reduces this by a factor of *r*, where *r* is the number of tokens to keep per group. The process is iterative and greedy:
-   **Step 1**: Divide the tokens into two disjoint sets.
-   **Step 2**: For each token in one set, find the most similar token in the other set using cosine distance between keys from self-attention.
-   **Step 3**: Keep only the top-*r* most similar edges between the two sets.
-   **Step 4**: Merge the endpoints of these edges via average pooling.

A key insight is the use of *proportional attention* to mitigate the issue of token aggregation. When *s* patches are merged into one token, the attention calculation is adjusted to account for the fact that the resulting token represents *s* original patches, preventing the aggregated token from being over-weighted.

### Program-of-Thoughts (PoT) Learning
The PoT learning strategy, described in Section 3.2, fundamentally changes the training objective. Instead of predicting a final answer, the model predicts a sequence of Python statements. The loss function, defined in Equation (2), is a language modeling loss over the response tokens, calculated only on the tokens in the generated program and its execution output.

The paper constructs the PoT dataset in two ways:
1.  **Template-based PoT**: Uses human-written templates with placeholders (e.g., `<legend-label>`, `<legend-values>`). The template is filled with data extracted from the chart, and the corresponding Python code is manually written to perform the calculation. This is shown in the upper half of Figure 4.
2.  **GPT-based PoT**: Leverages a smaller LLM (gpt-3.5-turbo) to generate the PoT answers. The model is prompted with the question, the chart's data table, and the instruction to generate a Python program. The generated code is then run through a Python interpreter to verify correctness. This is shown in the lower half of Figure 4.

### Training and Evaluation
The model is initialized from TinyLlama and uses SigLIP as the vision encoder and Phi-2 as the LLM. The training details are in Section 4.1: it runs for 3 epochs with a batch size of 512, a learning rate of 1e-4, and is trained on 32 Tesla V100 GPUs. The total dataset contains 1.36 million samples, as detailed in Table 1.

Evaluation is performed on four main benchmarks:
-   **ChartQA**: Measures accuracy for questions requiring numerical calculation.
-   **Chart-to-Text**: Measures BLEU-4 score for summarization.
-   **Chart-to-Table**: Measures RMSF1 score for table generation.
-   **OpenCQA**: Measures BLEU-4 for free-form QA.

The evaluation settings include Direct (direct answer), PoT (Python code), Combine (a hybrid of Direct and PoT), and Oracle (the best of Direct and PoT).

## Experimental Results
### Main Findings
As summarized in Table 2 and Section 4.3, TinyChart-3B achieves state-of-the-art performance on all major benchmarks while outperforming several 13B-parameter models.

-   **ChartQA**: TinyChart-3B achieves an overall accuracy of 83.60%, surpassing GPT-4V (78.50%), Gemini-Ultra (80.80%), and Qwen-VL-Max (79.80%). It also outperforms the previous SOTA open-source model, ChartAst (79.90%).
-   **Chart-to-Text**: TinyChart-3B achieves a BLEU-4 score of 17.93, significantly higher than the 13B models like Llama-1.5 (7.16) and Qwen-VL (61.60).
-   **Chart-to-Table**: TinyChart-3B achieves an RMSF1 score of 93.78, outperforming ChartAst (91.60) and ChartLlama (90.00).
-   **OpenCQA**: TinyChart-3B achieves a BLEU-4 score of 20.39, demonstrating superior performance in free-form question answering.

The paper highlights that TinyChart's success is due to the synergy of its two innovations. The PoT learning provides the model with robust numerical reasoning capabilities, while the VTM enables efficient processing of high-resolution charts, leading to faster inference and better performance.

### Inference Throughput
Figure 1(b) shows that TinyChart-3B achieves an impressive inference throughput of 3.14 it/s, which is significantly higher than the 1.94 it/s of ChartLlama-13B and the 1.47 it/s of ChartAst-13B. This demonstrates that the model is not only more accurate but also more efficient, making it suitable for real-time applications.

### Ablation Studies
The paper includes ablation studies to validate the contributions of each component. The results show that removing the PoT learning leads to a significant drop in performance on numerical tasks, confirming its importance. Similarly, removing the VTM module results in a degradation in performance, highlighting its role in enabling efficient high-resolution encoding.

In summary, the paper presents a compelling case for the effectiveness of combining efficient visual encoding with sophisticated reasoning strategies. TinyChart demonstrates that a small, efficient model can achieve state-of-the-art performance on complex chart understanding tasks, opening up new possibilities for deploying advanced AI capabilities in resource-constrained environments.
