# ChartCoder: Advancing Multimodal Large Language Model for Chart-to-Code Generation

## Core Ideas
The paper addresses a critical gap in multimodal large language models (MLLMs): while they excel at chart *understanding* via textual descriptions, they often fail at *chart-to-code* generation due to information loss and poor detail restoration. The core problem is that interpreting charts with text leads to oversimplification, whereas parsing charts into executable code provides a lossless, precise representation that captures all critical details—such as exact colors, coordinates, and parameter values—that are essential for reproducible data visualization.

The authors propose **ChartCoder**, the first dedicated chart-to-code MLLM, which leverages a Code LLM (DeepSeek Coder 6.7B) as its language backbone. This is a fundamental shift from prior work that used general-purpose LLMs or fine-tuned them on chart-text pairs. The key insight is that code is a more expressive and precise medium than natural language for representing complex visual structures.

To support this, the paper introduces two major innovations:
1.  **Chart2Code-160k**: The first large-scale, diverse dataset for chart-to-code generation, containing 160k high-quality chart-code pairs across 27 chart types.
2.  **Snippet-of-Thought (SoT)**: A novel training method that transforms direct generation into a step-by-step process, forcing the model to reason through each critical component of the chart before producing the final code.

The central motivation is that existing open-source MLLMs, despite their success in chart understanding, perform poorly on chart-to-code tasks because they lack the "code execution" capability and are not trained on sufficient, diverse chart-code data. ChartCoder aims to bridge this gap by combining a powerful code-generation backbone with a rich, structured dataset and a reasoning-focused training paradigm.

## Key Innovations
*   **First Dedicated Chart-to-Code MLLM**: ChartCoder is the first model specifically designed for the chart-to-code task, moving beyond the "chart-understanding-first" paradigm. It uses a Code LLM (DeepSeek Coder 6.7B) as its core language model, which is better suited for generating syntactically correct and executable code than a general-purpose LLM.

*   **Chart2Code-160k Dataset**: This is a landmark contribution, providing the first large-scale, diverse dataset for the task. Its scale (160k pairs) and diversity (27 chart types) directly address the scarcity of high-quality, code-aligned chart data that has been a bottleneck for previous research.

*   **Snippet-of-Thought (SoT) Method**: This is the paper's most significant methodological innovation. SoT restructures the generation process from a single, monolithic output to a multi-step, reasoning-driven pipeline. Instead of asking the model to generate the entire code at once, it breaks down the task into four logical steps: (1) Identify chart type and layout, (2) Generate data and colors, (3) Add critical details (e.g., `hatch`, `loc`), and (4) Produce the final, complete code. This forces the model to focus on and explicitly reason about the most important aspects of the chart, leading to higher accuracy and fewer errors like "Wrong Chart Type" or "Size mismatch."

*   **Any Resolution Strategy**: To handle the challenge of varying chart resolutions without losing fine details, the paper proposes an "Any Resolution" strategy. This involves resizing the input image to a pre-defined optimal aspect ratio, then dividing it into patches of standard resolution, which are concatenated with a downscaled version of the original image. This preserves both the overall structure and the fine-grained details necessary for accurate code generation.

*   **Two-Stage Training Paradigm**: The model is trained in two distinct stages: (1) Chart-to-text alignment, where the vision encoder and connector are frozen and the model learns to describe chart structures, and (2) Chart-to-code instruction tuning, where all components are jointly fine-tuned on the new Chart2Code-160k dataset and additional code-related data.

## Method & Implementation
The methodology is built around three pillars: dataset creation, model architecture, and a novel training process.

**1. Data Generation Pipeline (Figure 2a & Section 3.1/3.2):**
The paper outlines two distinct pipelines for creating the Chart2Code-160k dataset.

*   **Direct Generation:** An initial 200k set of code snippets is generated using a prompt that asks the LLM to create code for a given chart image. This is followed by a rigorous filtering process to remove problematic charts (e.g., those with excessive pixels or ticks) and ensure the generated code is executable.

*   **Step-by-Step Generation (SoT):** To enhance quality and reasoning, a subset of 50k chart-code pairs is sampled from the initial set. These are then processed through the Snippet-of-Thought method. For each pair, the model is prompted to generate the code in four sequential steps, with each step building upon the previous one. This creates a high-fidelity, multi-step instruction-tuning dataset.

**2. Model Architecture (Section 4.1):**
ChartCoder follows a standard MLLM architecture but with key modifications:
*   **Vision Encoder:** Uses SigLIP-384, a vision transformer pre-trained on a large web corpus.
*   **Vision-Language Connector:** A two-layer MLP that projects the visual features from the encoder into the word embedding space of the language model.
*   **Language Backbone:** A Code LLM, specifically DeepSeek Coder 6.7B, which is a state-of-the-art model for code generation. This is the core innovation, replacing the general-purpose LLM used in other MLLMs.

**3. Training Process (Section 4.2 & Figure 2b):**
The training is conducted in two stages:
*   **Stage 1: Chart-to-text Alignment:** The vision encoder and connector are frozen. The model is trained on a variety of chart-related corpora (UniChart, Chart-to-Text, SciCap, SciCap+) to learn how to describe chart structures in natural language. This aligns the visual and textual representations.
*   **Stage 2: Chart-to-code Instruction Tuning:** All components of the model are fine-tuned together on the newly created Chart2Code-160k dataset. This stage teaches the model to map the aligned visual and textual information directly to executable code, leveraging the power of the Code LLM backbone.

**4. Evaluation Metrics:**
The paper evaluates performance on several benchmarks, including ChartMimic, Plot2Code, and ChartX. The metrics include:
*   **Exec.Rate:** The percentage of generated code that is syntactically correct and can be executed.
*   **Low-Level / High-Level:** Scores for basic chart elements (e.g., axes, labels) vs. complex elements (e.g., legends, annotations).
*   **Pass Rate:** The percentage of test cases where the generated code produces a chart that matches the ground truth.
*   **Text-Match / Rating:** For specific benchmarks, scores are calculated based on the similarity between the generated code and the reference code, or a human-like rating system.

## Experimental Results
The experimental results, presented in Tables 3 and 4, demonstrate the effectiveness of the proposed approach.

**1. Overall Performance (Table 3):**
ChartCoder (Ours) achieves state-of-the-art results across all evaluated benchmarks. On the ChartMimic benchmark, it scores **91.4** on the full score, significantly outperforming all other open-source models (the next best is InternVL2-Llama3-76B at 83.2). On the Plot2Code benchmark, it achieves a remarkable **87.9** pass rate, far surpassing the second-best model (Qwen2-VL-72B) at 73.3. This confirms the paper's claim that ChartCoder outperforms existing open-source MLLMs on chart-to-code tasks.

**2. Detailed Analysis (Table 4):**
The detailed breakdown on the ChartMimic Direct Mimic task shows that ChartCoder's superiority is consistent across all sub-scores. It achieves the highest scores in every category: **16.83** for Chart Types, **9.13** for Layout, **14.77** for Text Content, **12.41** for Data, **12.68** for Style, and **8.29** for Clarity. This indicates that the model doesn't just generate correct code; it generates code that is also highly accurate in terms of the chart's structure, content, and aesthetic qualities.

**3. Ablation Studies (Implied by Section 4.2):**
While not explicitly detailed in the provided text, the paper's emphasis on the SoT method and the Any Resolution strategy suggests these were key ablation points. The results imply that the combination of the Code LLM backbone, the large-scale dataset, and the step-by-step training process is what enables the superior performance. The "Any Resolution" strategy is crucial for handling real-world images, and the SoT method is essential for preventing the common errors shown in Figure 1 (e.g., wrong chart type, size mismatch).
