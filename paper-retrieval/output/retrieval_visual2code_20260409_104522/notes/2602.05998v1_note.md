# VisRefiner: Learning from Visual Differences for Screenshot-to-Code Generation

## Core Ideas
The paper addresses the challenge of screenshot-to-code generation, where the goal is to translate a static UI screenshot into executable frontend code (HTML/CSS) that faithfully reproduces its layout and style. The core problem is that while multimodal large language models (MLLMs) can generate code directly from screenshots, they often lack precise visual alignment and structural fidelity.

The authors observe that human developers do not generate code in one step; instead, they iteratively render their implementation, compare it to the target design, and learn from the resulting visual discrepancies. This process provides rich, localized feedback signals that link appearance to implementation.

VisRefiner proposes a novel training framework that enables MLLMs to learn directly from these visual differences. It introduces a *difference-driven* paradigm where the model learns by observing discrepancies between its rendered output and the target design, rather than relying on one-way supervised mapping. The framework consists of two stages: (1) Difference-Aligned Supervision, which establishes an explicit correspondence between visual discrepancies and corrective code edits, and (2) Reinforcement Learning with Self-Refinement, where the model uses perceptual rewards to iteratively refine its own outputs.

## Key Innovations
*   **Difference-Driven Training Paradigm:** Unlike traditional one-way supervised learning, VisRefiner treats visual discrepancies as primary learning signals, enabling the model to understand how specific code changes affect visual outcomes.
*   **Difference-Aligned Supervision:** A novel data construction method that creates paired examples of "imperfect implementation" and "corrected counterpart," explicitly linking localized visual deviations to corresponding code edits. This provides fine-grained, structured supervision.
*   **Hybrid Perturbation Library:** A library of rule-based perturbations covering six visual dimensions (color, layout, alignment, component, image, text) that simulate real-world development inconsistencies, combined with model-predicted imperfections for richer, more diverse training data.
*   **Reinforcement Learning with Self-Refinement:** An integrated GRPO-based optimization stage that allows the model to generate, compare, and refine its own outputs, encouraging iterative reasoning and internalizing refinement behavior.
*   **Hierarchical Reward Design:** A composite reward structure with three components—Format, Refinement, and Quality—that ensures syntactic validity, visual improvement, and sensitivity to subtle perceptual differences.
*   **VisDiffUI Dataset & Pipeline:** A new dataset and data pipeline that aligns visual differences with corresponding code edits, providing a resource for studying appearance-to-implementation correspondences.

## Method & Implementation

### 3.1 Task Definition
The task is defined as learning a mapping from a target UI screenshot \(I_{gt}\) to structured frontend code \(C_{gen}\), such that rendering the generated code produces a result visually indistinguishable from the original design: \(C_{gen} = f_{gen}(I_{gt})\), \(\text{Render}(C_{gen}) \approx I_{gt}\).

The *Visual Refinement* task extends this by allowing the model to improve its own output. Given current code \(C_t\), its rendered image \(I_t = \text{Render}(C_t)\), and the ground-truth target \(I_{gt}\), the model generates an updated code \(C_{t+1}\) to minimize the visual gap.

### 3.2 Constructing Difference-Aligned Pairs
The core innovation lies in constructing a corpus of *difference-aligned pairs*. Each pair consists of:
1.  **A perturbed or model-predicted implementation** (\(C_t\)), representing an imperfect state.
2.  **Its corrected counterpart** (\(C_{t+1}\)), representing the improved state.
3.  **The target design** (\(I_{gt}\)).

The training objective is to predict the corrected code \(C_{t+1}\) given the imperfect state (\(I_t, I_{gt}, C_t\)), minimizing the negative log-likelihood.

#### (1) Rule-Based Perturbations
A library of 6 major categories of visual perturbations is designed to simulate common development errors:
*   **Color:** Alters palette tokens/gradations (e.g., hue shifts).
*   **Layout:** Rearranges grid/flex structures (e.g., misplacement, overlap).
*   **Alignment:** Introduces margin/offset shifts or misalignment.
*   **Component:** Removes/substitutes interface elements (e.g., sidebars, buttons).
*   **Image:** Adjusts aspect ratios or removes placeholders.
*   **Text:** Changes font size, weight, or truncates content.

These are applied to reference designs to create imperfect implementations, ensuring each modification is traceable to a specific code change.

#### (2) Model-Predicted Imperfections
To complement synthetic perturbations, the framework also incorporates outputs from an earlier-stage model. These "model-predicted imperfections" contain realistic artifacts like misplaced containers or missing tokens, creating a richer distribution of visual discrepancies.

### 4.1 Overview of VisRefiner
The framework operates in two distinct stages:

1.  **Stage 1: Difference-Aligned Supervision**
    *   Uses the constructed difference-aligned corpus to train the model via supervised learning.
    *   The model learns to associate localized visual differences with the correct code edits.
    *   This stage grounds the model's understanding of how implementation changes affect appearance.

2.  **Stage 2: GRPO with Self-Refinement**
    *   Applies Group Relative Policy Optimization (GRPO) to enable stable reinforcement learning.
    *   The model generates candidate code variants, renders them, and receives a composite reward based on visual improvement.
    *   The policy is updated to maximize the normalized advantage within each group of candidates.

### 4.2 Difference-Aligned Supervision
This stage establishes the explicit correspondence between observed visual discrepancies and required code updates. The loss function is the standard cross-entropy for sequence prediction:
\[
\mathcal{L}_{\text{SFT}} = -\log P_\theta(C_{t+1} \mid I_t, I_{gt}, C_t)
\]
where the target code \(C_{t+1}\) is generated autoregressively, conditioned on the visual comparison between the current rendered image \(I_t\) and the target \(I_{gt}\), as well as the context of the current code \(C_t\).

### 4.3 Reinforcement Learning with Self-Refinement
Building on the supervision stage, this stage enables the model to refine its own outputs through visual feedback.

#### Reward Design
A composite reward \(r_t\) is designed to guide the refinement process:
*   **Format reward (\(r_{\text{format}}\)):** Penalizes syntactically invalid HTML/CSS with -1, gives 0 otherwise.
*   **Refinement reward (\(r_{\text{improve}}\)):** Assigns +1 if the new rendering improves visual similarity (\(s_{t+1} > s_t\)), 0 otherwise.
*   **Quality reward (\(r_{\text{quality}}\)):** Uses the continuous Relative Improvement Score (RIS) to measure the degree of improvement.

The total reward is: \(r_t = r_{\text{format}} + r_{\text{improve}} + r_{\text{quality}}\).

#### Quantifying Visual Improvement
The Relative Improvement Score (RIS) evaluates the effectiveness of a refinement by measuring the relative progress in visual similarity:
\[
\text{RIS}(I_t, I_{t+1}, I_{gt}) = 
\begin{cases}
\frac{s_{t+1} - s_t}{1 - s_t}, & s_{t+1} > s_t \\
0, & \text{otherwise}
\end{cases}
\]
where \(s_t = \text{CLIP}(I_t, I_{gt})\) and \(s_{t+1} = \text{CLIP}(I_{t+1}, I_{gt})\). RIS captures relative progress across consecutive renderings, providing a stable and scale-invariant indicator.

#### Optimization with GRPO
The model employs Group Relative Policy Optimization (GRPO) to enable stable learning under visual feedback. For each target screenshot, the model generates a set of candidate implementations \(C^{(i)}\), renders them to obtain images \(I^{(i)}\), and computes a reward \(r_i\) for each variant using the formula above.

The normalized advantage is calculated as:
\[
\hat{r}_i = \frac{r_i - \bar{r}}{\sigma_r + \epsilon}
\]
where \(\bar{r}\) and \(\sigma_r\) are the mean and standard deviation of rewards within the group.

The GRPO objective is:
\[
\mathcal{L}_{\text{GRPO}} = -\frac{1}{G} \sum_{i=1}^G \min\left(\rho_i \hat{r}_i, \text{clip}(\rho_i, 1-\epsilon, 1+\epsilon) \hat{r}_i\right)
\]
where \(\rho_i = \frac{\pi_\theta(C^{(i)} \mid I_{gt}, I_t, C_t)}{\pi_{\theta_{\text{old}}}(C^{(i)} \mid I_{gt}, I_t, C_t)}\).

Each epoch begins by generating new single-step refinements from the current checkpoint, ensuring the model continually learns from its most recent predictions.

## Experimental Results

### Datasets
The authors construct **VisDiffUI**, a new dataset derived from Vision2UI and WebSight. It contains approximately 20K training instances, with samples divided into three partitions:
*   **SFT (Supervised Fine-Tuning):** Paired samples for supervised difference-aligned training (20K samples).
*   **RL(GRPO):** Unpaired samples for reinforcement learning (400 samples).
*   **EVAL:** Held-out evaluation set (100 samples).

### Evaluation Metrics
Five fine-grained metrics are used to evaluate layout fidelity, textual accuracy, spatial alignment, stylistic consistency, and perceptual similarity:
*   **Block Match:** Proportion of correctly aligned visual block regions.
*   **Text Accuracy:** Character-level Sørensen-Dice coefficient over matched text regions.
*   **Position Alignment:** Normalized spatial offset between corresponding block centers.
*   **Color Consistency:** Mean perceptual similarity measured by CIEDE2000 color difference.
*   **CLIP Similarity:** Perceptual alignment of masked renderings using CLIP-ViT-B/32 embeddings.

### Results

**Key Findings:**
*   **Significant Improvement with Self-Refinement:** VisRefiner-7B consistently outperforms its single-step version across all datasets, with improvements ranging from +0.3 to +5.2 points in AVG score.
*   **Superior Performance on Complex Benchmarks:** On the challenging Design2Code-HARD dataset, VisRefiner-7B achieves an AVG of 86.3, surpassing all other models including GPT-4o (85.6) and Qwen2.5-VL-72B (82.7).
*   **Strong Generalization:** On the held-out VisDiffUI-TEST set, VisRefiner-7B achieves an impressive AVG of 92.5, demonstrating strong generalization capability.
*   **Robustness Across Models:** The self-refinement stage benefits various base models, with VisRefiner-7B showing the largest gains (+0.7 to +5.2).

The results demonstrate that VisRefiner substantially improves single-step generation quality and layout fidelity, while also endowing models with strong self-refinement ability. The framework effectively leverages visual differences as learning signals, advancing the state-of-the-art in screenshot-to-code generation.
